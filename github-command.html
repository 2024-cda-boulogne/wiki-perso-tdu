GitHub

Git est un système de gestion de version distribuée largement utilisé dans le développement logiciel. Il permet de suivre l'évolution des fichiers d'un projet, de coordonner le travail collaboratif entre les membres d'une équipe et de revenir à des versions antérieures du code. Dans un projet Git, vous travaillerez généralement avec un dépôt central sur une plateforme telle que GitHub, GitLab, ou Bitbucket, tout en ayant une copie locale de votre code sur votre machine.


Voici donc les 10 commandes de git les plus utilisés :


Initialisation du Projet avec `git init` ou `git clone`

bash
git init
Utilisé pour créer un nouveau dépôt Git dans un répertoire existant sur votre machine.
bash
git clone URL_DU_DEPOT
Clone un dépôt existant depuis une URL, créant ainsi une copie locale du projet.

Ajout et Enregistrement des Modifications avec git add et git commit

bash
git add nom_fichier
Ajoute les modifications d'un fichier à la zone de préparation, signalant qu'elles seront incluses dans le prochain commit.
bash
git commit -m "Message de commit"
Enregistre les modifications ajoutées à la zone de préparation dans le dépôt avec un message de commit descriptif.

Gestion des Branches avec git branch et git checkout

bash
git branch nom_branche
Crée une nouvelle branche pour travailler sur une fonctionnalité ou une correction de bug
bash
git checkout nom_branche
Bascule entre les branches existantes pour travailler sur une branche spécifique

Fusion des Branches avec git merge

bash
git merge nom_branche
Fusionne une branche spécifiée dans la branche actuelle une fois le travail terminé

Synchronisation avec le Dépôt Central avec git pull et git push

bash
git pull origin nom_branche
Récupère les changements depuis le dépôt distant et fusionne avec la branche locale en cours.
bash
git push origin nom_branche
Pousse les modifications locales vers le dépôt distant, permettant ainsi la collaboration avec d'autres contributeurs.

Historique des Commits avec git log

bash
git log
Consulte l'historique des commits pour suivre l'évolution du projet, les auteurs, et les messages associés à chaque changement.
